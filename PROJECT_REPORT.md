# تقرير شامل عن مشروع نادي الأمن السيبراني (TUCC)

## 1. نظرة عامة على المشروع

### 1.1 الهدف والرؤية
مشروع **نادي الأمن السيبراني بجامعة الطائف (TUCC)** هو منصة تعليمية تفاعلية متكاملة مصممة لتعليم الأمن السيبراني بشكل عملي ومحاكاة. يهدف المشروع إلى:
- تعليم الطلاب المفاهيم النظرية والعملية في الأمن السيبراني
- توفير بيئة محاكاة آمنة للتدرب على الأدوات والهجمات
- بناء جيل من المحترفين في مجال الأمن السيبراني
- توفير محتوى تعليمي منظم ومحدث

### 1.2 الجمهور المستهدف
- طلاب الجامعات المهتمين بالأمن السيبراني
- المبتدئين في مجال Linux والشبكات
- المهتمين باختبار الاختراق والدفاع السيبراني
- الأعضاء المسجلين في النادي

## 4. الصفحات والمكونات الرئيسية

**الوصف:** صفحة الهبوط الرئيسية للنادي  
**المميزات:**

- خيارات الترتيب (الشعبية / التقييم / الأحدث)

## 5. محاكيات التدريب بالتفصيل

### 5.1 محاكي Bash الاحترافي (`BashSimulatorPro.jsx`)
**الوصف:** بيئة تفاعلية لتعلم أوامر Linux  

**الوحدات التعليمية (7 وحدات):**

| الوحدة | الدروس | الأوامر |
|--------|--------|---------|
| **الوحدة 1: الأساسيات** | 6 دروس | `pwd`, `ls`, `ls -a`, `cd`, `cd ..`, `clear` |
| **الوحدة 2: إدارة الملفات** | 6 دروس | `mkdir`, `touch`, `cp`, `mv`, `rm`, `man` |
| **الوحدة 3: القراءة والكتابة** | 4 دروس | `cat`, `echo`, `>`, `>>` |
| **الوحدة 4: الاستعراض** | 4 دروس | `head`, `tail`, `less`, `grep` |
| **الوحدة 5: إدارة الأجهزة** | 4 دروس | `df`, `du`, `mount`, `uname` |
| **الوحدة 6: إدارة العمليات** | 4 دروس | `ps`, `top`, `kill`, `pgrep` |
| **الوحدة 7: الصلاحيات والسكربتات** | 4 دروس | `chmod`, `chown`, `chmod 644`, `#!` |

**المميزات:**
- طرفية تفاعلية حقيقية (LTR)
- نظام ملفات افتراضي
- شاشة اختيار الوحدات مع عرض التقدم
- نظام نقاط XP للتحفيز
- شاشة نظرية لكل درس مع الشرح والتلميحات
- أوامر سريعة للوصول السريع

### 5.2 محاكي الهجمات - عملية الفهد الأسود (`AttackSimulatorPro.jsx`)
**الوصف:** سيناريو اختراق كامل (CTF)  

**مراحل العملية (5 مراحل):**
1. **الاستطلاع (Reconnaissance)** - جمع المعلومات
2. **المسح (Scanning)** - فحص المنافذ والخدمات
3. **الاستغلال (Exploitation)** - استغلال الثغرات
4. **ما بعد الاستغلال (Post-Exploitation)** - الحفاظ على الوصول
5. **التقرير (Reporting)** - كتابة التقارير

**الأوضاع:**
- **وضع المهاجم (Attacker):** منظور الهجوم (Red Team)
- **وضع المدافع (Defender):** منظور الدفاع (Blue Team)

**المميزات:**
- طرفية تفاعلية مع LTR
- سيناريو قصصي متكامل
- نظام Flags للتحدي
- نظام ملاحظات للمستخدم
- أوامر حقيقية: `whois`, `nmap`, `hydra`, `sqlmap`, `grep`, `tcpdump`, `iptables`

## 6. الواجهة الخلفية (Backend)

### 6.1 جداول قاعدة البيانات (SQLite)

| الجدول | الوصف |
|--------|-------|
| **users** | مستخدمي النظام (طلاب، مشرفين) |
| **simulators** | المحاكيات المتاحة |
| **user_progress** | تتبع تقدم المستخدمين |
| **content** | المحتوى التعليمي (مقالات، أخبار) |
| **system_logs** | سجلات النظام والأحداث |
| **activity** | تتبع أنشطة المستخدمين |

### 6.2 نقاط النهاية للـ API (Endpoints)

#### المصادقة (Authentication)
| النقطة | الطريقة | الوصف |
|--------|---------|-------|
| `/api/auth/login` | POST | تسجيل الدخول |
| `/api/auth/register` | POST | إنشاء حساب جديد |

#### الإدارة (Admin)
| النقطة | الطريقة | الوصف |
|--------|---------|-------|
| `/api/admin/stats` | GET | إحصائيات لوحة التحكم |
| `/api/admin/users` | GET | قائمة المستخدمين |
| `/api/admin/users/:id` | PUT | تحديث مستخدم |
| `/api/admin/users/:id` | DELETE | حذف مستخدم |
| `/api/admin/simulators` | GET/POST | إدارة المحاكيات |
| `/api/admin/content` | GET/POST | إدارة المحتوى |
| `/api/admin/logs` | GET | عرض السجلات |

#### المحاكيات (Simulators)
| النقطة | الطريقة | الوصف |
|--------|---------|-------|
| `/api/simulators` | GET | قائمة المحاكيات |
| `/api/simulators/:id` | GET | تفاصيل محاكي |

#### التقدم (Progress)
| النقطة | الطريقة | الوصف |
|--------|---------|-------|
| `/api/progress` | POST | تحديث التقدم |
| `/api/user/progress` | GET | تقدم المستخدم |

#### المحتوى (Content)
| النقطة | الطريقة | الوصف |
|--------|---------|-------|
| `/api/content` | GET | قائمة المحتوى |
| `/api/content/:id` | GET | تفاصيل المحتوى |

### 6.3 ميزات الواجهة الخلفية
- **JWT Authentication:** مصادقة آمنة باستخدام JSON Web Tokens
- **Role-based Access:** صلاحيات基于 الأدوار (admin, student)
- **Activity Tracking:** تتبع أنشطة المستخدمين
- **Real-time Polling:** تحديثات فورية للإحصائيات
- **File Upload:** دعم رفع الملفات مع شريط التقدم
- **API Caching:** نظام تخزين مؤقت للطلبات

## 7. السياقات والحالات (Contexts & State)

### 7.1 AnalyticsContext
**الوظيفة:** تتبع وتحليل سلوك المستخدم التعليمي  
**المكونات:**
- **الطبقات المعرفية (Cognitive Layers):**
  - Conceptual (مفاهيمي)
  - Operational (عملياتي)
  - Visual (بصري)
- **Skill Graph:** رسم بياني للمهارات المفتوحة والمغلقة
- **Ethical Compass:** بوصلة أخلاقية لقياس الميل الدفاعي/الهجومي
- **Learning Journey:** سجل رحلة التعلم مع الإنجازات والتحديات

### 7.2 DatabaseContext
**الوظيفة:** إدارة البيانات المحلية للعرض السريع  
**المميزات:**
- تخزين في LocalStorage للبيانات الدائمة
- عمليات CRUD كاملة للمستخدمين والمحتوى
- بيانات أولية (Seed Data) للاختبار

## 12. الخلاصة

مشروع نادي الأمن السيبراني هو منصة تعليمية متكاملة تجمع بين:
- **التعلم النظري:** عبر المقالات والدروس
